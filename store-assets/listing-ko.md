## 자세한 설명 (스토어 등록정보 → 설명, 한국어)

```
색각 이상 사용자가 보는 그대로 내 사이트를 확인하고, 그들만 겪는 대비 문제까지 찾아내세요.

dichroma는 과학적으로 정확한 색각 이상(CVD) 시뮬레이션을 어떤 페이지에든 실시간으로 적용합니다. 유형을 고르고 심각도 슬라이더를 움직이면, 페이지 전체가 적색맹(protanopia)·녹색맹(deuteranopia)·청황색맹(tritanopia)·전색맹(achromatopsia) 사용자가 지각하는 모습으로 다시 그려집니다. 이어서 내장 대비 검사를 실행하면 페이지의 텍스트를 '시뮬레이션된 색 공간'에서 다시 검사해, 일반 시력 기준 WCAG는 통과하지만 색각 이상에서는 읽기 어려워지는 텍스트 — 보통의 대비 검사기가 찾아낼 수 없는 문제 — 를 찾아냅니다.

dichroma가 다른 이유
• 과학적 정확성: 동료 검토를 거친 Viénot(1999)·Brettel(1997)·Machado(2009) 모델을 선형 RGB에서 올바르게 구현하고 DaltonLens 레퍼런스 데이터로 검증했습니다. 대충 만든 근사 필터가 아닙니다.
• 심각도 조절: 가벼운 이상 삼색형 색각부터 완전한 이색형 색각까지 0–100% 슬라이더로 조절합니다. Chrome DevTools의 내장 에뮬레이션은 고정된 최대 심각도만 제공합니다.
• 어디에도 없는 대비 검사: 글자색과 배경색을 모두 CVD 모델로 변환한 뒤 WCAG 대비를 다시 계산합니다. 대부분의 사람에게는 멀쩡히 읽히지만 남성 12명 중 1명꼴의 사용자에게는 읽히지 않는 빨강/초록 조합을 찾아냅니다.

주요 기능
• 5가지 시뮬레이션 모드: 적색맹·녹색맹·청황색맹·전색맹·끄기 (심각도를 낮추면 색약 변형)
• 심각도 슬라이더(0–100%), 실시간 반영
• '페이지 이동 후에도 유지' 옵션: 새로고침·이동 후 시뮬레이션 자동 재적용 (옵트인 시에만 호스트 권한 요청)
• 시뮬레이션 토글 단축키 지정 가능 (chrome://extensions/shortcuts)
• 사이드 패널 대비 검사 — 결과를 세 그룹으로 분류: '지금은 통과 — 색각 이상에서는 미달', '모든 사용자에게 이미 WCAG 미달', '검토 필요'
• 모든 결과에 원본 vs 시뮬레이션 미리보기 칩, 유형별 요약 카운트 제공
• 결과를 클릭하면 해당 요소로 스크롤·강조 표시되고 페이지 위 미리보기 카드가 열리며, 문제 요소는 오버레이로 표시
• 한국어·영어 UI (한국어 UI는 의학 명칭을 병기)

한계 (정직하게)
최상위 레이어 콘텐츠(열린 <dialog>, 전체화면 동영상)는 페이지 CSS 필터 밖이라 시뮬레이션되지 않습니다. chrome:// 페이지나 Chrome Web Store 등 제한된 페이지는 필터·검사가 불가능합니다. 대비 검사는 최상위 프레임만 검사하며 iframe 내부는 건너뜁니다. 시뮬레이션 색 공간의 대비 값은 CVD 색 모델에서 유도한 추정치이지 공식 WCAG 판정이 아니므로, 결과는 사람이 검토할 후보로 다뤄 주세요.

개인정보
dichroma는 어떤 데이터도 수집하지 않습니다. 모든 색 분석은 브라우저 안에서 로컬로 실행되며 네트워크 요청이 전혀 없습니다. 저장되는 것은 사용자 본인의 설정과 탭별 검사 결과뿐이고, 모두 사용자 기기의 Chrome 확장 프로그램 저장소에만 보관됩니다.
```
